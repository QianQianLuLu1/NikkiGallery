# i18n 补全计划

> 来源：P2-F3（`i18n/index.ts:4` 标注"机翻初版"未规划补全）

## 1. 现状

- **基准语言**：zh-CN（最完整，开发者维护）
- **支持语言**：12 种（zh-CN / zh-TW / en / ja / ko / fr / de / es / ru / pt / th / vi）
- **回退策略**：i18next `fallbackLng: 'zh-CN'`，缺失 key 自动回退
- **翻译来源**：除 zh-CN 外均为机翻初版，未经母语审校
- **覆盖率**：`npm run i18n:check` 报告约 45% key 在非基准语言中缺失（回退到 zh-CN）

## 2. 问题分类

| 级别 | 问题 | 影响 |
|------|------|------|
| P0 | 硬编码中文未走 i18n（已在 P2-U2/U3/F7 逐步修复） | 非中文用户无法理解 |
| P1 | 机翻不准确（术语不统一、语境缺失） | 体验差但不阻断使用 |
| P2 | key 缺失回退到 zh-CN | 非中文用户看到中文片段 |

## 3. 补全优先级

### 第一阶段：key 完整性（对应 P2-F7）
- 目标：所有语言文件 key 与 zh-CN 完全对齐
- 范围：PropertiesDialog / FeedbackDialog / ContextMenu / ShareCodeDecoderDialog 等 common 组件
- 验证：`npm run i18n:check` 零缺失

### 第二阶段：高频界面术语校对
- 目标：Gallery / Detail / Editor / Settings 四个高频页面的机翻校对
- 重点术语统一：
  - 套装 → Outfit（非 Suit）
  - 染色 → Dye（非 Color）
  - 分享码 → Share Code（非 Share Password）
  - 档案 → Profile（非 File）
  - 回收站 → Recycle Bin（非 Trash）

### 第三阶段：小语种母语审校
- 优先级：en > ja > ko > zh-TW > 其他
- 方式：社区贡献 / 专业翻译服务
- 验证：母语使用者验收

## 4. 维护规则

1. **新增 UI 文案必须同步添加 i18n key**（zh-CN + en 至少）
2. **禁止在组件中硬编码中文**（P2-U2/U3 已建立模式）
3. **`npm run i18n:check` 纳入 CI**（缺失 key 阻断合并）
4. **`npm run preview:check` 监控 preview.html 漂移**（P1-C1 已建立）
5. **翻译更新只修改 locale JSON，不改代码**
