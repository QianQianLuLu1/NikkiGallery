## 一、5 类高频问题排查

### 1. 启动失败

**现象**：双击图标后程序打不开，或弹出错误对话框。

#### 步骤 1：检查是否已有实例在运行

**现象**：弹出"应用已在运行中"对话框。

**操作**：
1. 看任务栏右下角系统托盘，是否有本程序图标
2. 如果有，双击图标激活已有窗口即可
3. 如果没有，点击对话框的**"清理并重启"**按钮（默认选项），程序会自动结束残留进程并重启
4. 如果清理失败，点击**"手动处理"**，在打开的任务管理器中结束所有名为"无限暖暖相册管理工具"的进程，再重新双击图标

#### 步骤 2：查看启动错误日志

**现象**：弹出"软件启动时遇到错误"对话框。

**操作**：
1. 对话框上点击**"打开日志目录"**按钮
2. 在打开的文件夹中找到 `startup-errors.log` 文件
3. 用记事本打开，查看里面的错误信息
4. 把这个文件发给我们（见[第四节](#四联系开发者前要准备什么)）

#### 步骤 3：清理可能的损坏文件

**现象**：启动时卡在加载界面，或反复闪退。

**操作**：
1. 按 `Win + R` 打开运行
2. 输入 `%APPDATA%\wxnn-photo-manager` 回车
3. 找到以下文件并删除（**注意：删除前先备份**）：
   - `wxnn_photo_manager.db-wal`
   - `wxnn_photo_manager.db-shm`
   - `wxnn_photo_manager.db-journal`
4. 不要删除 `wxnn_photo_manager.db`（这是你的媒体数据库）
5. 重新启动程序

#### 步骤 4：重置配置（最后手段）

**现象**：以上都无效，程序仍无法启动。

**操作**：
1. 按 `Win + R` 打开运行
2. 输入 `%APPDATA%\wxnn-photo-manager` 回车
3. 找到 `config.json` 文件，重命名为 `config.json.bak`
4. 重新启动程序（会用默认配置创建新的 config.json）
5. 如果启动成功，你可以删除 `.bak` 文件；如果启动失败，把 `.bak` 改回 `config.json` 还原

### 2. 扫描不到截图

**现象**：打开程序后看不到游戏截图，列表是空的。

#### 步骤 1：确认游戏已生成截图

**操作**：
1. 打开游戏，进入相册界面
2. 确认有照片存在
3. 关闭游戏（避免文件占用）

#### 步骤 2：手动触发扫描

**操作**：
1. 在程序中打开**设置**页
2. 找到"扫描"或"媒体管理"相关选项
3. 点击**"立即扫描"**或**"重新扫描"**按钮
4. 等待扫描完成（首次扫描较慢，可能需要几分钟）

#### 步骤 3：手动指定游戏目录

**现象**：自动扫描后仍然没有图片。

**操作**：
1. 在程序中找到"游戏目录"或"扫描路径"设置
2. 点击**"添加路径"**或**"自定义路径"**
3. 浏览并选择你的游戏安装目录，例如：
   - Steam 默认：`C:\Program Files\Steam\steamapps\common\Infinity Nikki`
   - Epic 默认：`C:\Program Files\Epic Games\InfinityNikki`
   - 自定义安装位置：找到包含 `InfinityNikki.exe` 或 `NikkiLauncher.exe` 的文件夹
4. 保存后再次触发扫描

#### 步骤 4：检查截图是否在云同步目录

**现象**：游戏目录正确，但扫描后图片数量不对。

**操作**：
1. 检查你的游戏截图是否在 OneDrive、iCloud、Dropbox 等云同步目录中
2. 如果是，**关闭云同步**或把截图**移动到非同步目录**
3. 因为系统会跳过云同步目录的扫描，避免重复
4. 重新扫描

#### 步骤 5：检查游戏安装类型

**现象**：以上都没用，扫描不到任何图片。

**操作**：
1. 确认游戏是**国服**还是**国际服**版本
2. 程序默认支持国服安装目录结构
3. 如果是国际服，请在设置中切换"游戏版本"或联系开发者支持

### 3. 图片不显示

**现象**：列表中有图片记录，但缩略图是灰色图标或破碎图标。

#### 步骤 1：检查文件是否还存在

**现象**：图片角标显示"已丢失"或"丢失"。

**操作**：
1. 这说明源文件已被移动或删除
2. 找到原始截图文件（通常在游戏目录的 `Saved\GamePlayPhotos` 下）
3. 把文件移回原位置，或重新扫描更新路径

#### 步骤 2：清空缩略图缓存

**现象**：文件存在，但缩略图加载不出来。

**操作**：
1. 打开**设置**页
2. 找到"缩略图管理"或"缓存管理"选项
3. 点击**"清空缓存"**按钮
4. 等待程序重新生成缩略图（首次显示会稍慢）
5. 滚动浏览图片，让程序逐个生成

#### 步骤 3：检查缓存目录权限

**现象**：清空缓存后仍然无法生成。

**操作**：
1. 打开**设置 → 日志与诊断 → 日志管理**
2. 点击**"打开目录"**按钮，查看当前 userData 路径
3. 检查 `%APPDATA%\wxnn-photo-manager\thumbnails\` 目录是否存在且可写
4. 如果目录不可写（被杀毒软件锁定等），右键文件夹 → 属性 → 安全 → 给当前用户添加"完全控制"权限
5. 或在设置中**修改缩略图缓存目录**到其他位置（如 `D:\thumbnails`）

#### 步骤 4：检查文件格式

**现象**：某些特定图片显示不出来。

**操作**：
1. 右键该图片，选择"打开所在文件夹"
2. 检查文件扩展名是否为 `.jpg` / `.png` / `.webp` / `.mp4`
3. 如果是非标准格式（如 `.heic`），需要先用其他工具转换
4. 如果是标准格式但无法显示，把文件路径记录下来反馈给开发者

### 4. 程序卡顿

**现象**：滚动列表卡顿、切换页面慢、操作响应延迟。

#### 步骤 1：等首次扫描完成

**现象**：刚打开程序时很卡。

**操作**：
1. 看程序底部或状态栏是否有扫描进度条
2. **等待扫描完全结束**再操作（首次扫描可能需要 5-10 分钟）
3. 扫描期间程序会大量读取磁盘，属于正常现象

#### 步骤 2：减少同时显示的图片数

**现象**：图片很多时滚动卡顿。

**操作**：
1. 在设置中找到"分页大小"或"每页显示数"
2. 调整为较小的值（建议 50-100 张/页）
3. 使用搜索/筛选功能缩小显示范围，避免一次显示几千张

#### 步骤 3：清理数据库

**现象**：使用一段时间后越来越卡。

**操作**：
1. 备份重要数据
2. 打开**设置 → 数据管理**或**数据库维护**
3. 点击**"清理无效记录"**或**"优化数据库"**
4. 等待操作完成（不要中途关闭程序）

#### 步骤 4：检查磁盘空间

**操作**：
1. 打开"此电脑"
2. 检查 C 盘剩余空间，建议**至少保留 10GB 可用空间**
3. 如果空间不足，清理临时文件：
   - 按 `Win + R`，输入 `%temp%` 回车，全选删除（跳过占用的文件）
   - 清空回收站
4. 重启程序

#### 步骤 5：关闭其他占用资源的程序

**操作**：
1. 关闭其他大型软件（特别是 Photoshop、视频编辑器等）
2. 关闭浏览器多余标签页
3. 在任务管理器中查看 CPU/内存占用高的程序，按需结束

#### 步骤 6：降低缩略图质量

**操作**：
1. 在设置中找到"缩略图质量"或"图像处理"
2. 切换到"低质量"或"性能模式"
3. 关闭"高清缩略图"选项（高分屏专用）

### 5. 崩溃闪退

**现象**：程序运行中突然关闭，或操作某个功能时崩溃。

#### 步骤 1：重启程序看是否复现

**操作**：
1. 重新双击图标启动程序
2. **记录导致崩溃的操作步骤**（点击了什么、做了什么）
3. 如果重启后正常，可能是偶发问题，继续使用观察
4. 如果每次操作都崩溃，按下面步骤处理

#### 步骤 2：导出崩溃报告

**操作**：
1. 重新启动程序
2. 打开**设置 → 日志与诊断 → 崩溃报告**
3. 看列表中是否有崩溃记录（按时间排序，最新在上）
4. 如果有记录，点击**"打开目录"**按钮
5. 在打开的文件夹中，把所有 `.dmp` 文件和对应的 `.txt` 文件打包成 ZIP
6. 把这个 ZIP 文件发给我们（见[第四节](#四联系开发者前要准备什么)）

#### 步骤 3：查看崩溃详情

**操作**：
1. 在**设置 → 日志与诊断 → 崩溃报告**列表中
2. 点击最新的一条崩溃记录展开
3. 查看"崩溃原因"和"调用栈"
4. 把这些信息截图或复制下来
5. 把信息连同操作步骤一起发给我们

#### 步骤 4：处理渲染进程崩溃

**现象**：弹出"页面已崩溃，需要重新加载"对话框。

**操作**：
1. 点击对话框的**"重新加载"**按钮（默认选项）
2. 如果重新加载后正常，继续使用
3. 如果反复崩溃，点击**"关闭应用"**，然后按步骤 2 导出崩溃报告

#### 步骤 5：处理运行时错误

**现象**：弹出"应用遇到错误"对话框，提示未预期的错误。

**操作**：
1. 对话框中提示"建议在设置→诊断中导出诊断包反馈给开发者"
2. 按对话框提示操作：
   - 打开**设置 → 日志与诊断 → 日志管理**
   - 点击**"导出"**按钮
   - 选择保存位置，把 ZIP 文件发给我们
3. 对话框只会在**首次**错误时弹出，后续会静默记录到日志，不会反复打扰
4. 如果想查看后续错误，去日志目录找对应的 `faults-YYYY-MM-DD.jsonl` 文件

#### 步骤 6：安全模式启动（最后手段）

**现象**：程序完全无法启动，反复崩溃。

**操作**：
1. 按 `Win + R`，输入 `%APPDATA%\wxnn-photo-manager` 回车
2. 重命名 `config.json` 为 `config.json.bak`（程序会用默认配置启动）
3. 重命名 `wxnn_photo_manager.db` 为 `wxnn_photo_manager.db.bak`（备份数据库）
4. 启动程序，如果能启动，说明配置或数据库损坏
5. 把 `.bak` 文件发给我们分析，并尝试用备份恢复数据

## 二、日志文件位置

### 默认位置

```
%APPDATA%\wxnn-photo-manager\logs\
```

**快速打开方法**：
1. 按 `Win + R`
2. 输入 `%APPDATA%\wxnn-photo-manager\logs` 回车

### 日志文件说明

| 文件名格式 | 内容 | 用途 |
|---|---|---|
| `main-YYYY-MM-DD.log` | 当天主日志 | 查看一般运行信息 |
| `faults-YYYY-MM-DD.jsonl` | 当天故障日志 | 查看错误和异常详情 |
| `startup-errors.log` | 启动错误日志 | 排查启动失败 |

**示例**：
- `main-2026-07-18.log` —— 2026 年 7 月 18 日的主日志
- `faults-2026-07-18.jsonl` —— 2026 年 7 月 18 日的故障日志

### 日志自动清理规则

- **总大小上限**：2GB（超出后自动删除最旧的日志）
- **保留时间**：90 天（超过自动删除）
- 你不需要手动清理，程序会自动维护

### 在程序中查看日志

1. 打开**设置**
2. 进入**"日志与诊断"**分组
3. 点击**"日志管理"**
4. 可以：
   - **查看故障列表**：自动加载，按时间排序
   - **打开目录**：用资源管理器打开日志文件夹
   - **清空**：删除所有日志（需二次确认）
   - **修改目录**：把日志存到其他位置（需重启程序）

### 修改日志目录

如果 C 盘空间不足或想集中管理：

1. 打开**设置 → 日志与诊断 → 日志管理**
2. 点击**"修改目录"**按钮
3. 选择新目录（如 `D:\wxnn-logs`）
4. 程序会自动迁移现有日志到新目录
5. **重启程序**生效

## 三、如何导出崩溃报告

### 方法 1：通过设置页导出诊断包（推荐）

诊断包包含**日志 + 崩溃报告 + 系统信息**，一次性收集完整。

**操作步骤**：

1. 打开**设置**
2. 进入**"日志与诊断"**分组
3. 点击**"日志管理"**
4. 点击**"导出"**按钮
5. 在弹出的保存对话框中：
   - 默认文件名：`wxnn-logs-YYYY-MM-DD.zip`
   - 选择保存位置（如桌面）
   - 点击"保存"
6. 等待导出完成（通常 5-30 秒，取决于日志大小）
7. 生成的 ZIP 文件就是诊断包

### 诊断包包含什么

| 文件/目录 | 内容 |
|---|---|
| `logs/` | 所有日志文件（`.log` 和 `.jsonl`） |
| `crashes/` | 所有崩溃 dump 文件（`.dmp` 等） |
| `startup-errors.log` | 启动错误日志 |
| `system-info.json` | 系统环境信息（操作系统、CPU、内存等） |
| `manifest.json` | 清单文件 |

**重要**：诊断包**不包含**你的媒体文件、数据库内容、个人账号信息，可以放心分享。

### 方法 2：手动复制崩溃文件

如果程序无法启动，无法通过设置页导出：

1. 按 `Win + R`
2. 输入 `%APPDATA%\wxnn-photo-manager\crashes` 回车
3. 把里面**所有文件**复制到一个文件夹
4. 再按 `Win + R`，输入 `%APPDATA%\wxnn-photo-manager\logs` 回车
5. 把里面**所有文件**复制到同一个文件夹
6. 把这个文件夹压缩成 ZIP

### 方法 3：导出单条故障详情

如果只想看某一条具体的错误：

1. 打开**设置 → 日志与诊断 → 日志管理**
2. 在故障列表中找到目标故障
3. 点击展开详情
4. 点击**"复制"**按钮，复制到剪贴板
5. 粘贴到聊天窗口或文档中发给我们

## 四、联系开发者前要准备什么

为了让我们快速定位问题，请准备好以下信息：

### 必备信息

1. **问题描述**：
   - 发生了什么？
   - 期望的结果是什么？
   - 实际的结果是什么？

2. **操作步骤**：
   - 你做了什么操作？
   - 按顺序列出每一步（例如：1. 打开程序 2. 点击"扫描" 3. 程序崩溃）

3. **诊断包**：
   - 按[第三节方法 1](#方法-1通过设置页导出诊断包推荐)导出诊断包
   - 这是最重要的材料，包含我们定位问题所需的所有信息

### 可选信息（有助于更快定位）

- **程序版本**：设置 → 关于 可以看到
- **操作系统版本**：按 `Win + R`，输入 `winver` 回车
- **截图/录屏**：如果问题涉及 UI，截图或录屏更直观
- **是否每次都能复现**：是 / 否 / 偶尔

### 联系方式

请通过以下方式联系我们：

- **GitHub Issue**（推荐）：[项目仓库地址]/issues
- **邮件**：[开发者邮箱]

发送时把诊断包作为附件，问题描述写在正文里即可。

## 附：常见错误提示对照表

| 错误提示 | 含义 | 应对方法 |
|---|---|---|
| "应用已在运行中" | 已有实例运行 | 点击"清理并重启" |
| "软件启动时遇到错误" | 启动失败 | 查看 startup-errors.log |
| "页面已崩溃，需要重新加载" | 渲染进程崩溃 | 点击"重新加载"，反复则导出崩溃报告 |
| "应用遇到未预期的错误" | 运行时异常 | 按提示导出诊断包 |
| "未能自动结束残留进程" | 单实例锁清理失败 | 手动打开任务管理器结束进程 |
| "全盘扫描未找到任何游戏媒体目录" | 找不到游戏 | 手动指定游戏目录 |
| "已丢失" / "丢失" 角标 | 文件被移动或删除 | 把文件移回原位置或重新扫描 |

> **小提示**：遇到问题先重启程序，能解决 80% 的偶发问题。剩下的 20% 按本手册操作，或导出诊断包联系我们。
