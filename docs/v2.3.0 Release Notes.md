# Infinity Nikki Gallery Manager v2.3.0 Release Notes

---

# 中文版

我们很高兴发布无限暖暖相册管理工具 v2.3.0。本次更新聚焦于扫描定位能力的全面强化、媒体来源的智能识别，以及退出稳定性和界面性能的深度打磨，带来更可靠、更流畅的图库管理体验。

## 新功能

### 媒体来源智能识别

新增 `media_source` 字段，自动区分三类媒体来源，图库可按来源快速过滤：

- **game**：游戏内拍照功能拍摄的照片
- **launcher**：启动器/游戏缓存素材（商城图 MallPic、F12 截图、云照片临时缓存等非用户拍摄内容）
- **cloud**：用户自己的云相册

### 游戏目录自动定位增强

扫描器的游戏目录查找能力全面升级，覆盖更多安装场景：

- 新增 Steam 注册表查询 + `libraryfolders.vdf` 解析，支持所有 Steam 库路径
- 新增 Epic Games 注册表查询，定位 Epic 安装的《无限暖暖》
- 新增用户数据目录检测（`%LOCALAPPDATA%\InfinityNikki`、`Documents\InfinityNikki`）
- 采用「浅层搜索（深度 4）→ 深度扫描（深度 15）」两级策略，先快速命中常见目录再深度扫描
- 界面文案「全盘扫描」更名为「自动定位游戏目录」，准确反映功能定位

### 全盘媒体签名扫描

新增 `fullScan` 模式：不依赖游戏可执行文件，直接在所有磁盘搜索媒体签名文件夹，支持多盘符并发扫描。首次启动自动执行全盘扫描，确保媒体目录不遗漏。

### 桌面快捷方式自动补建

应用启动时主动检查并补创建桌面快捷方式，解决 NSIS 覆盖安装等场景下快捷方式缺失的问题：

- 使用 Electron 原生 `shell.writeShortcutLink` 创建
- 支持注册表自定义桌面路径
- 已存在则跳过，避免重复创建
- 非阻塞异步执行，不影响启动速度

### 诊断包一键导出

设置 → 诊断中新增一键导出诊断包功能，包含日志、崩溃报告与系统环境信息，便于问题反馈。改用 Windows 10 1803+ 自带的 `tar.exe` 打包，彻底解决 PowerShell `Compress-Archive` 因执行策略拦截和通配符失效导致的导出失败。

## 改进与优化

### 扫描可靠性

- 跳过系统目录与云同步目录（OneDrive、iCloudDrive、Dropbox、Google Drive），避免重复扫描
- ScreenShot 文件夹检测增加路径上下文校验，防止非游戏目录误判
- 启动时自动修复旧数据：重算 `media_source='unknown'` 记录的字段，清理误扫描的非游戏目录文件

### 进程退出稳定性

重构退出清理流程，根治进程挂起与僵尸进程持锁问题：

- `before-quit` 加入 2 秒超时保护，超时强制退出
- 关键清理步骤使用 `Promise.race()` 超时兜底
- 诊断日志改用同步写入，避免异步 I/O 阻塞退出
- 单实例锁在所有清理完成后立即释放，所有退出路径均显式释放
- `uncaughtException` 处理器加锁，避免高频错误弹窗

### 界面性能

- 页面切换 FPS ≥ 55，包体积增量 ≤ 20KB gzip
- 页面切换改用轻量淡入动画（8px 右移 + 淡入，220ms），移除整页 View Transitions API
- 毛玻璃效果优化：移除 `soft-pink-luxury` 主题的 `saturate` 滤镜，修复动画结束后仍模糊的问题
- 虚拟列表不使用 motion，保留 keep-alive 机制

### 数据库优化

- 扫描 SQL 预编译语句缓存，重置时机随数据库连接变化管理
- `character_profiles` 昵称列新增索引，优化搜索性能
- `operation_history` 启动时自动清理 30 天前记录，防止无限增长

## 问题修复

- 修复进程退出时异步 I/O 阻塞事件循环导致进程挂起的问题
- 修复 `soft-pink-luxury` 主题 `saturate` 滤镜导致动画结束后画面仍模糊的问题
- 修复 800ms 等待竞态条件
- 修复僵尸进程持有单实例锁导致新实例启动失败的问题
- 进程检测由已弃用的 `wmic` 切换为 PowerShell `Get-CimInstance`，回退分支排除当前 PID 避免自杀
- 修复 PowerShell `Compress-Archive` 执行策略拦截与 `-LiteralPath` 通配符失效导致的诊断包导出失败

# English Version

We are pleased to release Infinity Nikki Gallery Manager v2.3.0. This update focuses on comprehensive enhancements to scan-and-locate capabilities, intelligent media source recognition, and deep refinements to exit stability and UI performance—delivering a more reliable and smoother gallery management experience.

## New Features

### Intelligent Media Source Recognition

A new `media_source` field automatically classifies media into three sources, with quick gallery filtering by source:

- **game**: Photos taken via the in-game photo mode
- **launcher**: Launcher/game cache assets (MallPic storefront images, F12 screenshots, cloud photo temp cache—non-user content)
- **cloud**: The user's own cloud album

### Enhanced Game Directory Auto-Location

The scanner's game directory lookup is significantly upgraded to cover more installation scenarios:

- Added Steam registry query + `libraryfolders.vdf` parsing to support all Steam library paths
- Added Epic Games registry query to locate Epic-installed *Infinity Nikki*
- Added user data directory detection (`%LOCALAPPDATA%\InfinityNikki`, `Documents\InfinityNikki`)
- Two-tier strategy: shallow search (depth 4) → deep scan (depth 15), hitting common directories first
- UI text "Full Disk Scan" renamed to "Auto-Locate Game Directory" to accurately reflect the feature

### Full-Disk Media Signature Scan

A new `fullScan` mode locates media signature folders across all drives without relying on game executables, with concurrent multi-drive scanning. The first launch automatically performs a full scan to ensure no media directory is missed.

### Desktop Shortcut Auto-Creation

The app now proactively checks and creates the desktop shortcut on startup, resolving missing shortcuts in scenarios such as NSIS overwrite installs:

- Created via the native Electron `shell.writeShortcutLink`
- Supports custom desktop paths configured via the registry
- Skipped if already present to avoid duplicates
- Non-blocking async execution with no impact on startup speed

### One-Click Diagnostic Package Export

A new one-click diagnostic package export is available in Settings → Diagnostics, bundling logs, crash reports, and system environment info for easier issue reporting. It uses the built-in `tar.exe` (Windows 10 1803+) instead of PowerShell `Compress-Archive`, eliminating export failures caused by PowerShell execution-policy blocks and wildcard issues.

## Improvements & Optimizations

### Scanner Reliability

- Skips system directories and cloud-sync directories (OneDrive, iCloudDrive, Dropbox, Google Drive) to avoid duplicate scanning
- Adds path-context validation for ScreenShot folder detection to prevent false positives from non-game directories
- Auto-repairs legacy data on startup: recalculates fields for `media_source='unknown'` records and cleans up mistakenly scanned non-game files

### Process Exit Stability

The exit cleanup flow is reworked to eliminate process hangs and zombie-process lock holding:

- `before-quit` now has a 2-second timeout guard with forced exit on timeout
- Critical cleanup steps are wrapped in `Promise.race()` timeout fallbacks
- Diagnostic logs use synchronous writes to prevent async I/O from blocking exit
- The single-instance lock is released immediately after all cleanup, explicitly across every exit path
- The `uncaughtException` handler is guarded to prevent high-frequency error popups

### UI Performance

- Page-switch FPS ≥ 55, bundle size increment ≤ 20KB gzip
- Page transitions use a lightweight fade animation (8px right shift + fade in, 220ms); whole-page View Transitions API removed
- Frosted glass optimization: removed the `saturate` filter on the `soft-pink-luxury` theme, fixing persistent blur after animation
- Virtual lists do not use motion; keep-alive mechanism preserved

### Database Optimization

- Scanner SQL prepared statements are cached and reset in sync with database connection changes
- Added an index on the `character_profiles` nickname column to optimize search performance
- `operation_history` auto-cleans records older than 30 days on startup to prevent unbounded growth

## Bug Fixes

- Fixed process hangs caused by async I/O blocking the event loop during exit
- Fixed persistent blur after animation in the `soft-pink-luxury` theme caused by the `saturate` filter
- Fixed an 800ms wait race condition
- Fixed new-instance startup failures caused by zombie processes holding the single-instance lock
- Switched process detection from the deprecated `wmic` to PowerShell `Get-CimInstance`, with the fallback branch excluding the current PID to avoid self-termination
- Fixed diagnostic package export failures caused by PowerShell `Compress-Archive` execution-policy blocks and `-LiteralPath` wildcard issues
